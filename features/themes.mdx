## Theme structure

Each theme includes six color properties:

```swift
// From Theme.swift:3-12
struct AppTheme: Identifiable, Equatable {
    let id: Int
    let name: String
    let mainBg: Color          // Primary background
    let secondaryBg: Color     // Cards, panels, secondary surfaces
    let border: Color          // Borders, dividers
    let text: Color            // Primary text
    let secondaryText: Color   // Labels, subtitles
    let accent: Color          // Highlights, charts, buttons
}
```

<Info>
  All theme colors are defined using RGB values for precise color matching across different displays.
</Info>

## Changing themes

### From the dashboard

1. Click the theme button in the header (shows current theme name and color dot)
2. A popover opens with all 24 themes organized into Light and Dark sections
3. Hover over any theme to preview it in real-time
4. Click to apply permanently

```swift
// From DashboardView.swift:153-172
private var themeSelector: some View {
    Button {
        showThemePicker.toggle()
    } label: {
        HStack(spacing: 8) {
            Circle().fill(previewTheme.accent).frame(width: 8, height: 8)
            Text(previewTheme.name).font(.system(size: 10, weight: .bold))
        }
        .frame(width: 120, alignment: .center)
        .background(previewTheme.secondaryBg)
        .clipShape(RoundedRectangle(cornerRadius: 8))
    }
    .popover(isPresented: $showThemePicker) {
        themePickerPopover
    }
}
```

<Tip>
  The entire dashboard updates instantly when you hover over a theme, letting you preview before committing.
</Tip>

## Theme persistence

```swift
// From DashboardView.swift:37
@AppStorage("app_theme_id") private var appThemeId = 0
```

Your theme choice is saved using `@AppStorage`, which:
- Persists across app launches
- Syncs automatically with UserDefaults
- Applies immediately on app startup

<Accordion title="Default theme behavior">
  If no theme is saved, TypeSteps defaults to theme ID `0` (Light theme).
  
  The app automatically applies the correct ColorScheme (`.light` or `.dark`) based on the theme:
  
  ```swift
  // From DashboardView.swift:71-96
  private let lightThemeIds = [0, 12, 13, 14, 15, 17, 18, 19, 20, 21, 22, 23]
  
  .colorScheme(lightThemeIds.contains(theme.id) ? .light : .dark)
  ```
</Accordion>

## Creating custom themes

While TypeSteps doesn't support user-created themes in the UI, you can modify the source code to add your own:

```swift
// From Theme.swift:13-257
static let themes: [AppTheme] = [
    AppTheme(
        id: 24, // Use next available ID
        name: "My Custom Theme",
        mainBg: Color(red: 0/255, green: 0/255, blue: 0/255),
        secondaryBg: Color(red: 20/255, green: 20/255, blue: 20/255),
        border: Color(red: 40/255, green: 40/255, blue: 40/255),
        text: .white,
        secondaryText: Color(red: 150/255, green: 150/255, blue: 150/255),
        accent: Color(red: 255/255, green: 100/255, blue: 100/255)
    ),
    // ... existing themes
]
```

<Info>
  Remember to add your custom theme's ID to the `lightThemeIds` array if it's a light theme, so the correct ColorScheme is applied.
</Info>
